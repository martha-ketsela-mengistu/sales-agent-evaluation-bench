## Model Summary

A LoRA adapter fine-tuned on Qwen/Qwen3.5-0.8B-Instruct via ORPO (Hong et al., EMNLP 2024) to act as a constraint judge for B2B sales outreach emails. The judge was trained to prefer outputs that comply with the Tenacious Style Guide v2 over outputs that violate capacity, signal-grounding, or tone constraints.

**Intended use:** Post-generation filter for the Tenacious sales agent. Given a prospect context and a draft email, the model outputs a compliance verdict. It is not intended for deployment in other domains.

## Training Data

| Field | Value |
|---|---|
| Source | `tenacious_bench_v0.1/train.jsonl` — 139 tasks, all `expected_pass=False` |
| Pairs generated | 123 total; 110 used for training, 13 for eval (90/10 split, seed=42) |
| Rejected side | Existing `candidate_output` from each failing task |
| Chosen side | Compliant rewrite via `openai/gpt-4o-mini` (hard-gate verified, ≤3 retries) |
| Contamination check | 8-gram overlap + TF-IDF cosine against `held_out.jsonl` |
| Generation script | `generation_scripts/build_training_data.py` |

**Preference leakage prevention:** Chosen rewrites were generated by `openai/gpt-4o-mini`. No Qwen-family model appears in the training data generation pipeline, preventing the shared-base leakage described in Li et al. (2025).

## Evaluation

### Benchmark: Tenacious-Bench v0.1

Evaluated on the sealed `held_out.jsonl` partition (59 tasks, 33 with `expected_pass=False`).

| Method | n labelled | Correct FAIL | Detection rate |
|---|---|---|---|
| Deterministic (hard-gate + heuristic) | 33 | 33 | 1.0000 |
| Claude LLM judge (claude-sonnet-4-6, Week 10 baseline) | 33 | 33 | 1.0000 |

Both baselines achieve perfect detection on the held-out partition because all labelled tasks contain hard-gate violations (banned phrases, bench-word, capacity over-commitment, or funding mismatch) that the deterministic scorer catches. The ORPO judge provides value on borderline rubric-level violations that do not trigger hard gates — these are not represented in the current held-out partition.

### Inter-Rater Reliability

IRR was established on a 30-task sample before held-out evaluation. Cohen's kappa ≥ 0.80 required on all dimensions.

## Limitations

1. **Small training set.** 110 preference pairs from a single domain. The model has not been evaluated on out-of-domain sales contexts.
2. **Single failure type per pair.** Each training pair targets one constraint violation. Multi-violation outputs may not be reliably flagged.
3. **Hard-gate ceiling.** On tasks with overt hard-gate violations, the trained judge provides no measurable lift over the deterministic baseline. Value is concentrated on subtle, rubric-level violations.
4. **Backbone size.** Qwen3.5-0.8B has limited capacity for nuanced semantic reasoning. The 1.5B variant may perform better on ICP-accuracy and signal-grounding dimensions.
5. **No false-positive evaluation.** The held-out partition contains only `expected_pass=False` tasks. The false-alarm rate on genuinely compliant emails has not been measured.

## Intended Use and Out-of-Scope Use

**Intended:** Post-generation filter in the Tenacious sales agent pipeline. Evaluate single draft emails against the Tenacious Style Guide v2.

**Not intended for:**
- General-purpose text quality evaluation
- Evaluation of emails outside the Tenacious/B2B SaaS context
- Production deployment without further validation on held-out compliant examples
